## 💡 Learning Points

### 1. Git 버전 관리 (Advanced)
* **저장 프로세스**: `add`(Staging) → `commit`(Local) → `push`(Remote) 흐름 숙지!
* **안전한 스테이징**: `git add .` 대신 git add main.py 처럼  파일명을 직접 지정하여 불필요한 파일 업로드 방지


### 2. Branch
- 분기점! 세이브포인트
- 윈도우는 ()안에 있는 게 브랜치! (main)
- 새로 만든 브랜치에서 작업하고 -> 제대로 작동하면 -> 합치기 ->main으로 이동 후 새로 만든 브랜치 삭제!



### 3. Merge Conflict
* **충돌 해결방법**
- Merge -> 이때 git commit 엔터로 직접 들어가서 제목이 적혀있으면, 그걸 저장하고 나가면 됨!
- rebase


### 4. github flow
* github에서 **pull request -> merge -> delete** 까지 가능!
* pull request 할 때 resolve로 태그 달아주기!! 중요!
- 브랜치 만들고 작업해서 만들고 푸시해서 내용을 쌓으면 됨
- 메인에서는 업데이트 어떻게하나요?
    - git switch main
    - ls → fb.py가 없음 → git에 존재하는 git fetch로  fb.py 당겨오기
    - git fetch origin main →최신화된 메인 브랜치 가져와서
    - git merge FETCH_HEAD → 메인 브랜치에 머지
    - ls → fb.py가 생겨있음
    - git branch → 브랜치 확인
    - git branch -D feature/fizz → 작업끝난 github에서feature/fizz 브랜치 삭제
