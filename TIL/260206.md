## 1. 🚀 효율적인 프로젝트 관리 (Issue, Label, Milestone)
팀 협업 시 무엇을(Issue), 어떤 성격으로(Label), 언제까지(Milestone) 할지 정의하는 것이 핵심입니다.

### 🏷️ Issue & Label & Milestone
| 개념 | 비유 | 주요 목적 | 예시 |
| :--- | :--- | :--- | :--- |
| **Issue** | 개별 할 일 | 작업 내용 기록 및 소통 | "FizzBuzz 함수 구현하기" |
| **Label** | 분류용 꼬리표 | 이슈의 성격과 긴급도 표시 | `feat`, `bug`, `high-priority` |
| **Milestone** | 마감 기한 (달력) | 프로젝트 주기 및 목표 관리 | "v1.0 배포 (2/10까지)" |

- **Label**: `Type`(Bug, feat), `Priority`(Urgent, Low), `Status`(In Progress)로 구분.
- **Milestone**: 스프린트(Sprint) 단위로 이슈를 묶어 전체 진척도(%) 파악.

## 3. 🔄 GitHub Flow 실전 프로세스
- Issue 생성: 팀 레포에 작업 내용 등록 (Description, Tasks, References 포함).

- Fork & Clone: 팀 레포를 내 계정으로 포크 후 로컬로 가져오기.

- Branch 생성: feature/fb 등 작업용 브랜치 생성.

- Pull Request (PR): 작업 완료 후 origin에 푸시하고 팀 레포로 PR 전송.

- Merge & Sync: 팀장이 머지하면 upstream에서 최신 소식 fetch 후 동기화.

## 4. 🛠️ Git 위기 탈출 및 고급 명령어
- 🧺 git stash: 작업 내용 임시 보관
커밋하기엔 미완성이지만 브랜치를 급히 이동해야 할 때 사용.

- git stash (보관) -> git stash pop (다시 꺼내기).

## ⏪ 수정 및 되돌리기 (Undo)
- git restore: 파일 수정 내용을 마지막 커밋 상태로 되돌림 (삭제 주의).

- git commit --amend: 직전 커밋 메시지 재작성 (아이디 새로 생성됨).

- git revert: 잘못을 인정하고 되돌리는 커밋을 새로 남김 (협업 권장).

- git reset: 이력을 지워버림. 개인정보 유출 등 특수 상황에서만 제한적 사용.
